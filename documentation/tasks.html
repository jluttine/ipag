Tasks sharing

The project can be split in two areas: game engine and UI. Task sharing was bit of a problem since there is three of us. Luckily, the windowing system is so flexible that different windows can be easily distributed to different people and AI and sound are independent but important areas. Workload was not completely balanced but close enough.

We kept no record of hours spent to the project, but the amout of code should give some idea. Note that people have written some code into other people's classes and some classes have even changed owner. Also, some people (meaning Jaakko) use more empty lines than others.

Communication

We had a meeting when ever there was an important decision to make that reqired conversation and careful planning. For smaller matters we used email and ICQ instant messenger, which turned out to be exremely useful. Cooperation was no problem at all.

Schedule

Schedule turned out to be pretty realistic. We had plenty of time for testing, fine tuning and bug fixes.
